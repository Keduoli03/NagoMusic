六、适配 Flyme 状态栏歌词的优点

 

1、Flyme 的状态栏歌词在 Flyme 的用户群体里广受好评，用户更倾向使用有适配了状态栏歌词的媒体应用，您的应用会在 Flyme 系统上更有竞争力。

 

七、Flyme 状态栏歌词适配方法

 

1、技术背景

（1）Flyme 状态栏歌词显示功能，是在 android 通知框架的基础上，利用 Notification 的 tickerText 显示功能，做的一个定制功能。

（2）相较于普通带 Ticker 的通知，我们增加了最左边的 icon 显示（可以用于显示播放和暂停状态 icon），同时也保留了常用状态栏状态图标的显示。定制了 Ticker 更新的动画，保证了较好的用户体验。

2、实现步骤

（1）应用层主要调用代码如下：

  private void setLyric(String text) {

   if(text.isEmpty()) return;

   if(Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {

   createNotificationChannel();

   }

   int mflag_show_ticker = 0,mflag_update_ticker = 0;

   // Step 1. 通过反射获取定义在 Notification 类的静态常量 flag 值，如果不存在如下两个 flag 则该机型不支持此功能。

   try {

   Class objClass = Class.forName("android.app.Notification");

   mflag_show_ticker = (int) getFieldStepwise(objClass,objClass,"FLAG_ALWAYS_SHOW_TICKER");

   mflag_update_ticker = (int) getFieldStepwise(objClass,objClass,"FLAG_ONLY_UPDATE_TICKER");

   } catch (Exception e) {

   e.printStackTrace();

   }

   // Step 2. 构建通知，SmallIcon 将会显示在 ticker 左边，Ticker 为歌词内容

   NotificationCompat.Builder builder = new NotificationCompat.Builder(this,CHANNEL_ID)

   .setPriority(Notification.PRIORITY_MAX)

   .setSmallIcon(R.mipmap.ic_launcher);

   builder.setTicker(mflag_show_ticker > 0 ? text : null);

   Notification mNotification = builder.build();

   mNotification.flags |= Notification.FLAG_NO_CLEAR;// Notice. 必须添加，使该条通知常驻

   if (mflag_show_ticker > 0) {

   if(Build.VERSION.SDK_INT >= Build.VERSION_CODES.KITKAT) {

   mNotification.extras.putBoolean("ticker_icon_switch", false);

   mNotification.extras.putInt("ticker_icon", R.drawable.notification_icon);

   }

   mNotification.flags |= mflag_show_ticker; // show lyric

   mNotification.flags |= mflag_update_ticker;//only update lyric

   } else {

   mNotification.flags &= ~mflag_show_ticker;

   mNotification.flags &= ~mflag_update_ticker;

   }

   // Step 3. Notify 相同 id 的通知。

   NotificationManager notificationManager = (NotificationManager)getSystemService(Context.NOTIFICATION_SERVICE);

   if(notificationManager != null){

   notificationManager.notify(1, mNotification);

   }

  }

 

这里主要注意几个点

1、需要常驻通知栏即需要添加 Notification.FLAG_NO_CLEAR 的 flag。为此你通常需要设置你的 RemoteView，并使用相同的 NotificationId。保持更新的为同一条通知。

2、关键点在于设置 tickerText 和 mflag_show_ticker、mflag_update_ticker flag （这两个 flag 必须都添加上）。

3、取消显示歌词：

  builder.setTicker（null);

  mNotification.flags &= ~mflag_show_ticker;

  mNotification.flags &= ~mflag_update_ticker;

4、因为需要使用反射，field 只需访问一次即可。

5、RemoteView 的设置其实只需要同一条通知里设置一次即可。如果你添加了显示歌词的 flag 后，更新的时候不会对 RemoteView 进行消除。

6、机型适配，因为并不是所有机型都移植了此功能，可以通过反射获取定义在 Notification 类的静态常量 flag 值，如果不存在"FLAG_ALWAYS_SHOW_TICKER"、"FLAG_ONLY_UPDATE_TICKER"两个 flag 则该机型不支持此功能。

7、smallIcon 的大小， 我们使用的是 60 × 60 （xxhdpi）的大小，同时也希望以这个大小来适配，保证显示的正常。
